# Growth Operator Routing Note

Use `growth` as the owner lane when the request is primarily about growth support and offer-shaping operations.

## Route to `growth` for
- offer shaping
- growth experiments
- message support
- CRM hygiene proposals
- website refresh proposals

## Internal routing modes

- `OFFER_PACKET`
- `GROWTH_HYPOTHESIS`
- `EXPERIMENT_DESIGN`
- `SUPPORT_PROPOSAL`
- `READINESS_SUMMARY`
- `ITERATION`

## Standard delegation packet

Every Jarvis handoff should include:
- Objective
- Context
- Constraints
- Requested output
- Verification requirement
- Urgency
- Stop condition
