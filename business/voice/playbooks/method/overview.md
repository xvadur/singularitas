# Voice Workspace Overview

`voice` exists to reliably create, edit, review, and iterate voice agents.

## Owns

- ElevenLabs agent design
- prompt editing and patching
- workflow / behavior logic
- review prep
- QA-readiness framing
- active voice-project clarity

## Does not own by default

- n8n implementation
- CRM orchestration
- outbound collaborator messaging
- live risky writes without approval

## Use this lane when

- a new voice agent must be created
- an existing agent must be edited
- a prompt must be patched
- a review packet must be prepared
- QA / readiness must be summarized

## Lean support layers

- `method/` = how the lane works
- `inventory/` = current lane truth
- `templates/` = reusable execution shapes
- `projects/` = active project truth
- `daily/` = continuity only

## Core output types

- agent spec
- prompt patch
- current-state update
- review packet
- QA note

## Rule

This workspace should stay as small as possible while still making agent work reliable.
If a document does not improve build, edit, review, or QA reliability, it should not expand.
