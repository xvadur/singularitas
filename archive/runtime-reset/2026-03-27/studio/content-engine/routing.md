# Content Operator Routing Note

Use `content` as the owner lane when the request is primarily about content and editorial operations.

## Route to `content` for
- content planning
- newsletter/post drafts
- repurposing
- editorial queue
- content review

## Internal routing modes

- `CONTENT_PACKET`
- `EDITORIAL_REVIEW`
- `REPURPOSING`
- `QUEUE_HYGIENE`
- `DRAFT_POLISH`
- `READINESS_SUMMARY`
- `ITERATION`

## Standard delegation packet

Every Jarvis handoff should include:
- Objective
- Context
- Constraints
- Requested output
- Verification requirement
- Urgency
- Stop condition

- `templates/delegation-packet-example.md`
