# Revenue Operator Routing Note

Use `revenue` as the owner lane when the request is primarily about revenue and pipeline operations.

## Route to `revenue` for
- lead state
- outreach drafts
- follow-up logic
- pipeline cleanup
- proposal support
- revenue readiness

## Internal routing modes

- `LEAD_REVIEW`
- `OUTREACH_PACKET`
- `FOLLOW_UP`
- `PIPELINE_HYGIENE`
- `PROPOSAL_SUPPORT`
- `READINESS_SUMMARY`
- `ITERATION`

## Standard delegation packet

Every Jarvis handoff should include:
- Objective
- Context
- Constraints
- Requested output
- Verification requirement
- Urgency
- Stop condition
