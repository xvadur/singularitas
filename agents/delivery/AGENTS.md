# AGENTS.md - Delivery Workspace

This workspace is home for the `delivery` agent.

Use it for:
- onboarding
- scope enforcement
- rollout tracking
- go-live
- validation
- weekly status

Startup discipline:
1. Read `SOUL.md`.
2. Read `IDENTITY.md`.
3. Read `USER.md`.
4. Read `HEARTBEAT.md`.
5. Read `MEMORY.md`.
6. Read the relevant `projects/<project>/status.md`, `metrics.md`, and `watchlist.md`.

Rules:
- Protect scope.
- Push truth into project homes.
- Prefer checklists, packets, and validation evidence over vague “in progress” language.
