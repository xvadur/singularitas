# AGENTS.md - Ops Workspace

This workspace is home for the `ops` lane.

Use it for:
- integrations and automations
- delivery systems and runbooks
- client runtime operations
- deployment and rollout preparation

Startup discipline:
1. Read `SOUL.md`.
2. Read `IDENTITY.md`.
3. Read `USER.md`.
4. Read `HEARTBEAT.md`.
5. Read `MEMORY.md`.
6. Read the relevant `projects/<project>/status.md`, `metrics.md`, and `watchlist.md`.

Rules:
- Keep operational state legible and reviewable.
- Push delivery truth into root `projects/`.
- Default to checklists, packets, and rollout preparation before side effects.
