# Runtime Tree 2026

Canonical Jarvis runtime tree:

- `control/` -> founder cockpit, tasks, habits, roadmaps, approvals, dashboards
- `personal/` -> mail, calendar, commitments, admin, relationships
- `business/` -> CRM, revenue, frontdesk, web, content, voice, automation
- `projects/` -> initiative-specific execution only
- `docs/` -> doctrine, architecture, playbooks, schemas, plans
- `memory/` -> daily continuity and curated support memory
- `data/` -> raw sync state, imports, normalized machine-readable data
- `outputs/` -> reviewable artifacts and evidence
- `systems/` -> runtime glue and automation code
- `skills/` -> curated live skills only
- `archive/` -> retired and legacy material

Non-rules:

- `agents/` is not a bootstrap source for `main`
- `firma` is not canonical truth for Jarvis
- `singularitas_opus` is not canonical truth for Jarvis
- `projects/` should not absorb canonical task, CRM, or roadmap state
