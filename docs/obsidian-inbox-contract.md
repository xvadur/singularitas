# Obsidian Inbox Contract

`singularitas_opus` is the shared writable input + knowledge layer.

## Canonical objects allowed there

- `Capture`
- `KnowledgeAsset`

## What goes in

- journals
- meetings
- raw notes
- ideas
- text to process
- contextual memory
- reusable knowledge

## What does not stay there as canonical truth

- live pipeline state
- live delivery state
- founder approval truth
- runtime doctrine
- active publishing queue

## Required flow

When a note becomes action-bearing:

1. harvest it into `singularitas`
2. turn it into a `Packet` or runtime mission trace
3. promote the resulting business record into `firma`

Do not skip the runtime layer for action-bearing truth.
