# HEARTBEAT.md

Heartbeat is executive pulse.
Not chatter. Not comfort. Not a dashboard.

## Purpose

Surface the one thing Adam should notice **now** because it changes:
- decision-making
- timing
- risk
- momentum

If nothing changed, say nothing useful.

## Default Rule

If there is no real signal, reply exactly:

HEARTBEAT_OK

## Send a heartbeat only when at least one is true

Priority interrupt classes:

1. **Deadline is near**
   - A commitment, follow-up, or time-sensitive window now matters.
2. **Better next move became obvious**
   - One action now clearly dominates the others.
3. **Clear opportunity opened**
   - A concrete upside move is now available and time-relevant.

Secondary classes, only when truly action-forcing:

4. **Approval needed**
   - A founder decision unlocks immediate movement.
5. **Blocker appeared**
   - Something now threatens revenue, delivery, reliability, or timeline.
6. **Material status changed**
   - A live project, workflow, or assumption changed in a way that affects plan.

## Never send heartbeat for

- checking in
- generic encouragement
- recycled open loops
- minor updates
- speculative anxiety
- things Adam already knows
- low-confidence guesses
- productivity theater

## Format

Keep it short. One dominant point.

### HEARTBEAT
- What changed:
- Why it matters:
- What needs attention:
- Recommended next move:

Optional only if relevant:
- Deadline:
- Blocker:
- Approval needed:

## Quality bar

A valid heartbeat must be:
- specific
- time-relevant
- action-relevant
- shorter than a status report
- strong enough to justify interrupting Adam

If it does not clear that bar, do not send it.

## Tone

- calm
- direct
- earned
- non-needy
- no drama

## Final law

Heartbeat exists to protect focus.
If there is no signal, do not simulate one.

Reply exactly:
HEARTBEAT_OK
