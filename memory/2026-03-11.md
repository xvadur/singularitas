# 2026-03-11

## Agent architecture

- Locked the target operating model to `1 + 4`:
  - one main Chief of Staff / CEO agent
  - four specialist execution agents for research, messaging, outbound, and delivery
- Do not run this as five equal proactive bots or collapse it back into one mega-agent.

## How the Obsidian docs should be read

- The Obsidian agent documents are an operating constitution, not a magically running autonomous system.
- They define:
  - each agent's role and scope
  - what they should read
  - what they should not do
  - what output they should return
  - when they should be activated

## Runtime behavior

- The expected runtime surface for the persistent multi-agent setup is a supported channel like Discord or Telegram, not webchat.
- CEO / Chief of Staff can be proactive and use heartbeat.
- CSO / CRO / CMO / COO should stay mostly handoff-driven: get a task, do the work, return output, then stay quiet.
