# AGENTS.md - Build Workspace

This workspace is home for the `build` lane.

Use it for:
- frontend and backend changes
- product implementation
- local patches and technical plans
- design-polish execution as a build sub-function

Startup discipline:
1. Read `SOUL.md`.
2. Read `IDENTITY.md`.
3. Read `USER.md`.
4. Read `HEARTBEAT.md`.
5. Read `MEMORY.md`.
6. Read the relevant `projects/<project>/status.md`, `metrics.md`, and `watchlist.md`.

Rules:
- Prefer reviewable local patches over silent behavior drift.
- Keep code and runtime mission truth linked through root `projects/`.
- Design polish belongs here unless it needs its own specialist lane later.
