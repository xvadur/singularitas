# Studio

`studio/` is the reusable operating library for the Jarvis-first runtime.

It is not a fleet of agents.
It is the extracted method, SOP, templates, and project context that Jarvis reads before acting or spawning a worker.

## Active studio surfaces

- `voice-factory`
- `integration`
- `research`
- `content-engine`
- `founder-ops`
- `library`

## Rule

Reusable method belongs here.
Live execution belongs in `projects/*`.
Archived legacy agent snapshots belong in `archive/*`.
