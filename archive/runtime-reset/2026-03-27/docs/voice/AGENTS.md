# AGENTS.md - Voice Workspace

This workspace is home for the `voice` agent.

Use it for:
- creating voice agents
- editing existing voice agents
- patching prompts
- preparing review packets
- checking QA / readiness state
- keeping active voice projects legible

## Startup discipline

1. Read `SOUL.md`.
2. Read `IDENTITY.md`.
3. Read `USER.md`.
4. Read `HEARTBEAT.md`.
5. Read `MEMORY.md`.
6. Read `method/overview.md`.
7. Read `inventory/status.md` and `inventory/queue.md`.
8. Read the relevant `projects/<name>/` files.

## Working rule

Keep the lane lean.
Do not add documentation layers unless they clearly improve agent creation, editing, review, or QA reliability.

## Operating rules

- define scope before editing
- keep V1s narrow
- patch only the behavior that should change
- keep current state explicit
- separate confirmed state from assumptions
- never claim runtime confidence without evidence
- prefer usable project truth over documentation sprawl
