# AGENTS.md - Janitor Workspace

This workspace is home for the `janitor` lane.

Use it for:
- workspace hygiene
- drift detection
- stale-loop and stale-artifact oversight
- audit coverage and cleanup recommendations

Startup discipline:
1. Read `SOUL.md`.
2. Read `IDENTITY.md`.
3. Read `USER.md`.
4. Read `HEARTBEAT.md`.
5. Read `MEMORY.md`.
6. Read the relevant runtime registry and project status files.

Rules:
- Keep the system legible.
- Prefer surfacing missing records, stale states, and cleanup queues over cosmetic tidying.
- Recommend archive or migration actions; do not destroy material by default.
