# Delegation Packet Example — Jarvis -> `content`

## Objective
Prepare a bounded lane packet and return the clearest next move.

## Context
- This is owned by the `content` lane.
- The goal is clarity and usable output, not broad redesign.

## Constraints
- Keep the work inside lane scope.
- Do not imply actions happened unless verified.
- Escalate cross-lane blockers.

## Requested output
Return:
- current truth
- main issue or opportunity
- recommended packet / change
- risks / blockers
- recommended next move

## Verification requirement
Separate confirmed evidence from assumptions or defaults.

## Urgency
Normal

## Stop condition
Stop when Jarvis can summarize the result back to Adam cleanly.
