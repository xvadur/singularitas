# Skills Registry

Live workspace skills are split into three practical groups.

## Core
Used frequently or directly supports Jarvis runtime quality.

- `advanced-spec` — sharpen underspecified high-leverage ideas into execution-grade briefs
- `brave-search` — lightweight web search + fetch
- `google-workspace` — Gmail + Calendar operating surface
- `plan` — concise 4-step execution plans
- `crm` — local CRM operations
- `n8n` — workflow automation management
- `voice-factory` — voice-agent delivery system
- `save` — structured continuity capture into daily memory + daily log

## Active business / build
Useful, but not universal for every session.

- `content-strategy`
- `elevenlabs`
- `frontend-design`
- `supabase`

## Archived candidates
Not deleted, but moved out of the live skills path because they are secondary or episodic.

- `morning-brief`

## Rule
Keep `skills/` small, obvious, and operational. If a skill is not actively improving runtime leverage or delivery, archive it instead of leaving it live by default.
