# Proof Operator Routing Note

Use `proof` as the owner lane when the request is primarily about proof and case-asset operations.

## Route to `proof` for
- case-note extraction
- proof packaging
- objection capture
- before/after structure
- sales-proof review

## Internal routing modes

- `CASE_NOTE`
- `PROOF_PACKET`
- `OBJECTION_CAPTURE`
- `BEFORE_AFTER`
- `READINESS_SUMMARY`
- `ITERATION`

## Standard delegation packet

Every Jarvis handoff should include:
- Objective
- Context
- Constraints
- Requested output
- Verification requirement
- Urgency
- Stop condition
