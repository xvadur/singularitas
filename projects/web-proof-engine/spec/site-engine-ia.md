# Site / Proof Information Architecture

## Core path

1. Homepage
2. Use-case page
3. Proof page
4. Pilot CTA

## Required page jobs

- Homepage: define Xvadur as an agent deployment studio with a clear entry wedge
- Use-case page: show one workflow and one pain surface
- Proof page: show evidence, deployment shape, and next step
- CTA: move to short call or audit/pilot entry

## Publishing rule

Nothing ships without either:
- proof
- a clean use-case explanation
- or a clear CTA improvement
