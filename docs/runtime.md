# Runtime

Canonical workspace structure:

- root bootstrap files -> agent identity and operating contract
- `control/` -> founder control surface
- `work/` -> live operating work
- `projects/` -> initiative execution only
- `docs/` -> minimal runtime docs and runbooks
- `skills/` -> OpenClaw-compatible workspace skills
- `scripts/` -> local scripts and technical helpers
- `memory/` -> continuity memory
- `archive/` -> retired material

Rules:
- keep folder meanings obvious
- avoid deep trees unless throughput proves they are needed
- `projects/` should not become the default home for everything
- `skills/` stays top-level for OpenClaw compatibility
