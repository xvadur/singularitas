# Pilot Workflow

## V1 flow

1. Prospect identified
2. Short call booked or direct founder conversation happens
3. Pilot fit confirmed
4. Pilot proposal packet prepared
5. Deployment packet prepared
6. Voice runtime configured
7. Callback / follow-up contract defined
8. Validation checklist run
9. Expansion considered only after validation

## Required objects

- `Lead`
- `Call`
- `Proposal`
- `Deployment`
- `Validation`

## Approval gates

- pricing promise
- client-facing proposal send
- production runtime changes
- expansion beyond one problem surface
